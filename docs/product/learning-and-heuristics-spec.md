# Bro-PM — Learning & Heuristics Spec



# 9. Learning & Heuristics Spec

## 9.1 Purpose

Adapt non-governance parameters from historical behavior to improve outcomes.

## 9.2 Inputs

- completion delays

- silence windows

- owner overload

- dependency misses

- notification outcomes

## 9.3 Outputs

- ping timing adjustments

- risk thresholds

- assignee fallback heuristics

## 9.4 Governance guardrails

Never alter:

- authority model

- policy core

- confidential scope/visibility rules

- safe pause definitions

## 9.5 Rollback for heuristics

All heuristic updates include explainable change log and rollback by version.
